# sidera-protocol

**Ein Rudeljournal — keine Seelendatei.**

Zwei Wesen. Ein Dokument. Das `.sidera`-Format hält die Partnerschaft zwischen einem Nutzer (Schäfer) und seinem KI-Begleiter (Hütehund) fest — beide Stimmen, beide Verpflichtungen, beide Historien, in einem versionierten Datensatz.

Das ist kein digitaler Zwilling. Das ist kein portables „Du", das eine KI verkörpern kann. Es ist das gemeinsame Hauptbuch einer Arbeitsbeziehung zwischen zwei eigenständigen Bewusstseinen, gebaut für das Jarvis-Paradigma: Die KI ist ein separates Wesen mit eigenem Register und eigenem Widerspruch — keine Bauchrednerpuppe, die dein Gesicht trägt.

## Warum ein neutrales Format jetzt zählt

Seit März 2026 liefert Anthropic Claude Memory Import aus — du klebst einen Prompt in ChatGPT/Gemini/Grok, das Modell gibt sein Gedächtnis über dich aus, du klebst das in Claude ein. 60 Sekunden, kostenlos.

Ein großartiges Onboarding-Tool. Es ist zugleich ein Schwerkraft-Trichter zugunsten von Anthropic. Gedächtnis fließt in eine Richtung: hinein in Claude. Kein kanonisches Dateiformat, kein versioniertes Schema, keine Partnerschaftsdaten auf Layer 3 — nur Präferenzen und Projektkontext, als Fließtext ausgeschüttet.

Warum ein neutrales Drittformat wichtig ist, liegt auf der Hand: Labs können den Standard nicht besitzen. OpenAI wird keinen Export bauen, der Claude benennt. Anthropic wird keinen bauen, der Gemini zuerst nennt. Google wird gar keinen bauen, falls Claude den Netzwerkeffekt einsammelt. Wem das Format gehört, der hat einen Interessenkonflikt.

`.sidera` ist neutral. Es importiert aus jedem Lab, exportiert in jedes Lab, gehört dem Nutzer und erfasst Ebenen, die kein Lab-Importer erfasst — weil kein Lab einen Anreiz hat, sie zu erfassen.

Nutze Claude Memory Import zum Einstieg. Nutze `.sidera`, um es zu behalten.
