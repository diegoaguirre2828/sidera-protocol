# sidera-protocol

**Журнал стаи — а не файл души.**

Два существа. Один документ. Формат `.sidera` фиксирует партнёрство между пользователем (пастухом) и его ИИ-спутником (овчаркой) — оба голоса, оба обязательства, обе истории в одной версионируемой записи.

Это не цифровой двойник. Это не портативный «ты», которого ИИ может надеть на себя. Это общая бухгалтерская книга рабочих отношений между двумя отдельными сознаниями, созданная для парадигмы Jarvis: ИИ — это самостоятельное существо со своим регистром и способностью возразить, а не кукла чревовещателя с твоим лицом.

## Почему нейтральный формат важен именно сейчас

По состоянию на март 2026 года Anthropic выпустила Claude Memory Import — вставляешь промпт в ChatGPT/Gemini/Grok, модель выгружает всё, что помнит о тебе, а ты вставляешь это в Claude. 60 секунд, бесплатно.

Отличный инструмент онбординга. И одновременно — гравитационная воронка в сторону Anthropic. Память течёт в одну сторону: внутрь Claude. Ни канонического формата файла, ни версионированной схемы, ни данных партнёрства уровня 3 — только предпочтения и контекст проектов, вылитые сплошной прозой.

Причина, почему важен нейтральный сторонний формат, проста: лаборатории не могут владеть стандартом. OpenAI не станет строить экспорт, в котором упомянут Claude. Anthropic не построит такой, где Gemini стоит первым. Google не построит вовсе, если сетевой эффект соберёт Claude. У того, кому принадлежит формат, есть конфликт интересов.

`.sidera` нейтрален. Он импортирует из каждой лаборатории, экспортирует в каждую лабораторию, принадлежит пользователю и фиксирует слои, которые не фиксирует ни один лабораторный импортёр — потому что ни у одной лаборатории нет стимула их фиксировать.

Используй Claude Memory Import для онбординга. Используй `.sidera`, чтобы сохранить это навсегда.

Полная спецификация, схемы, примеры и англоязычный README — на [github.com/diegoaguirre2828/sidera-protocol](https://github.com/diegoaguirre2828/sidera-protocol).
