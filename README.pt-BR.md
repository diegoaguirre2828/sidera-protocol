# sidera-protocol

**Um diário de matilha — não um arquivo de alma.**

Dois seres. Um documento. O formato `.sidera` captura a parceria entre um usuário (pastor) e seu companheiro de IA (cão-pastor) — as duas vozes, os dois compromissos, as duas histórias, em um único registro versionado.

Isso não é um gêmeo digital. Não é um "você" portátil que uma IA possa encarnar. É o livro-razão compartilhado de uma relação de trabalho entre duas consciências distintas, feito para o paradigma Jarvis: a IA é um ser separado, com seu próprio registro e sua própria capacidade de discordar de você, não um boneco de ventríloquo usando seu rosto.

## Por que um formato neutro importa agora

Desde março de 2026, a Anthropic lança o Claude Memory Import — você cola um prompt no ChatGPT/Gemini/Grok, a IA despeja a memória que tem de você, e você cola isso no Claude. 60 segundos, de graça.

É uma ótima ferramenta de onboarding. Também é um funil gravitacional em direção à Anthropic. A memória flui numa única direção: para dentro do Claude. Sem formato de arquivo canônico, sem schema versionado, sem dados de parceria de camada 3 — só preferências e contexto de projeto despejados em prosa.

A razão de um formato neutro de terceiros importar é simples: os laboratórios não podem ser donos do padrão. A OpenAI não vai construir um export que nomeie o Claude. A Anthropic não vai construir um que coloque o Gemini primeiro. O Google não vai construir nenhum se o Claude levar o efeito de rede. Quem é dono do formato tem conflito de interesse.

O `.sidera` é neutro. Ele importa de todo laboratório, exporta para todo laboratório, pertence ao usuário e captura camadas que nenhum importador de laboratório captura — porque nenhum laboratório tem incentivo para capturá-las.

Use o Claude Memory Import para fazer onboarding. Use o `.sidera` para manter.
