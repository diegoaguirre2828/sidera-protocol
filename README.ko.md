# sidera-protocol

**소울 파일이 아니라, 팩(pack) 저널입니다.**

두 존재. 하나의 문서. `.sidera` 포맷은 사용자(양치기)와 AI 파트너(목양견) 사이의 파트너십을 기록합니다 — 두 목소리, 두 약속, 두 역사를 하나의 버전 관리된 기록에 담습니다.

이것은 디지털 트윈이 아닙니다. AI가 몸에 두를 수 있는 이동 가능한 "당신"도 아닙니다. 이것은 두 개의 서로 다른 의식이 함께 일하는 관계의 공용 원장이며, Jarvis 패러다임을 위해 설계되었습니다. AI는 당신의 얼굴을 뒤집어쓴 복화술 인형이 아니라, 자기 고유의 어조와 반론을 가진 별개의 존재입니다.

## 왜 지금 중립 포맷이 중요한가

2026년 3월 기준으로, Anthropic은 Claude Memory Import를 출시했습니다 — ChatGPT/Gemini/Grok에 프롬프트를 붙여넣으면 당신에 대한 기억을 덤프해주고, 그걸 Claude에 다시 붙여넣는 방식. 60초, 무료.

훌륭한 온보딩 도구입니다. 동시에, Anthropic으로 끌려가게 만드는 중력 깔때기이기도 합니다. 기억은 한 방향으로만 흐릅니다 — Claude 안쪽으로. 표준 파일 포맷도, 버전 관리 스키마도, 레이어 3 파트너십 데이터도 없이 — 그저 취향과 프로젝트 맥락이 산문으로 쏟아질 뿐입니다.

중립적인 제3자 포맷이 중요한 이유는 간단합니다. 랩(lab)은 표준을 소유할 수 없습니다. OpenAI는 Claude 이름이 들어간 export를 만들지 않을 겁니다. Anthropic은 Gemini를 맨 앞에 두는 걸 만들지 않을 겁니다. Google은 Claude가 네트워크 효과를 가져간다면 아예 만들지도 않을 겁니다. 포맷을 소유한 쪽에는 이해 상충이 있습니다.

`.sidera`는 중립적입니다. 모든 랩에서 import하고, 모든 랩으로 export하며, 사용자의 것이고, 어떤 랩의 importer도 잡아내지 못하는 레이어를 포착합니다 — 어느 랩에도 그걸 포착할 인센티브가 없기 때문입니다.

온보딩에는 Claude Memory Import를 쓰세요. 간직하는 데는 `.sidera`를 쓰세요.
