# sidera-protocol

**一份群体日志 —— 不是灵魂文件。**

两个存在。一份文档。`.sidera` 格式记录用户（牧羊人）与其 AI 伙伴（牧羊犬）之间的协作关系 —— 两种声音、两份承诺、两段历史，都收录在一份有版本控制的记录中。

这不是数字孪生。这不是 AI 可以附身的便携式"你"。这是两种独立意识之间工作关系的共享账本，为 Jarvis 范式而建：AI 是一个独立的存在，拥有自己的语气与反驳能力，而不是戴着你面孔的腹语木偶。

## 为什么中立格式在此刻至关重要

截至 2026 年 3 月，Anthropic 推出了 Claude Memory Import —— 把一段提示词粘进 ChatGPT/Gemini/Grok，它会转储对你的记忆，你再把结果粘到 Claude。60 秒，免费。

这是一个出色的上手工具，但同时也是一个把用户引力吸向 Anthropic 的漏斗。记忆只流向一个方向：流进 Claude。没有标准的文件格式，没有版本化的 schema，没有第三层的伙伴关系数据 —— 只有以散文形式倾倒出来的偏好与项目上下文。

中立的第三方格式之所以重要，道理很简单：实验室无法拥有标准。OpenAI 不会构建一个把 Claude 写进去的导出工具。Anthropic 不会构建一个把 Gemini 放在首位的工具。Google 如果 Claude 赢下网络效应，就根本不会去做。格式归谁所有，谁就有利益冲突。

`.sidera` 是中立的。它从每个实验室导入，向每个实验室导出，归用户所有，并捕获任何实验室导入器都不捕获的层级 —— 因为没有一家实验室有动机去捕获它们。

用 Claude Memory Import 上手。用 `.sidera` 长期保存。

完整规范、schema、示例与英文 README 见 [github.com/diegoaguirre2828/sidera-protocol](https://github.com/diegoaguirre2828/sidera-protocol)。
