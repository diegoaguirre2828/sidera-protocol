# sidera-protocol

**群れの航海日誌であって、魂ファイルではない。**

ふたつの存在。ひとつのドキュメント。`.sidera` フォーマットは、ユーザー（羊飼い）とそのAIパートナー（牧羊犬）のあいだのパートナーシップを記録する —— 両者の声、両者のコミットメント、両者の歴史を、ひとつのバージョン管理された記録にまとめる。

これはデジタルツインではない。AIが憑依できるポータブルな「あなた」でもない。これはふたつの独立した意識のあいだにある協働関係の共有台帳であり、Jarvisパラダイムのために設計されている。AIはあなたの顔を被った腹話術人形ではなく、自分自身の口調と反論を持つ、別個の存在である。

## なぜ今、中立フォーマットが重要なのか

2026年3月時点で、AnthropicはClaude Memory Importをリリースした —— ChatGPT/Gemini/Grokにプロンプトを貼りつけると、あなたに関する記憶がダンプされ、それをClaudeに貼り直すだけ。60秒、無料。

優れたオンボーディングツールだ。しかし同時に、Anthropicへユーザーを吸い寄せる重力ファネルでもある。記憶は一方向にしか流れない —— Claudeへ、だ。標準ファイルフォーマットもなければ、バージョン管理されたスキーマも、レイヤー3のパートナーシップデータもない —— あるのは散文としてダンプされた好みとプロジェクト文脈だけだ。

中立な第三者フォーマットが重要な理由は単純だ。ラボは標準を所有できない。OpenAIはClaudeの名前が入ったエクスポート機能を作らない。AnthropicはGeminiを最初に置くものを作らない。Googleは、Claudeがネットワーク効果を取ったら、そもそも作らない。フォーマットを所有する者には利益相反がある。

`.sidera` は中立だ。あらゆるラボからインポートし、あらゆるラボへエクスポートし、ユーザー自身のものであり、どのラボのインポーターも拾わない層を捕らえる —— どのラボにもそれを拾うインセンティブがないからだ。

Claude Memory Importはオンボーディングに使え。`.sidera` は保持するために使え。

仕様、スキーマ、サンプル、英語版READMEの全文は [github.com/diegoaguirre2828/sidera-protocol](https://github.com/diegoaguirre2828/sidera-protocol) にある。
