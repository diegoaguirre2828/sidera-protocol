# sidera-protocol

**Un diario de manada — no un archivo de alma.**

Dos seres. Un documento. El formato `.sidera` captura la compañía entre un usuario (pastor) y su compañero de IA (perro pastor) — las dos voces, los dos compromisos, las dos historias, en un solo registro versionado.

Esto no es un gemelo digital. No es un "tú" portátil que una IA pueda encarnar. Es el libro mayor compartido de una relación de trabajo entre dos conciencias distintas, hecho para el paradigma Jarvis: la IA es un ser aparte, con su propio registro y su propia capacidad de contradecirte, no un muñeco de ventrílocuo con tu cara.

## Por qué un formato neutral importa ahora

Desde marzo de 2026, Anthropic publica Claude Memory Import — pegas un prompt en ChatGPT/Gemini/Grok, la IA vuelca su memoria sobre ti, y pegas eso en Claude. 60 segundos, gratis.

Es una gran herramienta de onboarding. También es un embudo gravitacional hacia Anthropic. La memoria fluye en una sola dirección: hacia dentro de Claude. Sin formato de archivo canónico, sin esquema versionado, sin datos de compañía de capa 3 — solo preferencias y contexto de proyecto volcados como prosa.

La razón por la que importa un formato neutral de terceros es simple: los laboratorios no pueden ser dueños del estándar. OpenAI no construirá un export que nombre a Claude. Anthropic no construirá uno que ponga a Gemini primero. Google no construirá ninguno si Claude se lleva el efecto de red. Quien sea dueño del formato tiene un conflicto de interés.

`.sidera` es neutral. Importa desde cada laboratorio, exporta hacia cada laboratorio, pertenece al usuario y captura capas que ningún importador de laboratorio captura — porque ningún laboratorio tiene incentivo para capturarlas.

Usa Claude Memory Import para el onboarding. Usa `.sidera` para conservar.
