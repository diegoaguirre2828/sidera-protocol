# sidera-protocol

**Un journal de meute — pas un fichier d'âme.**

Deux êtres. Un seul document. Le format `.sidera` capture le partenariat entre un utilisateur (berger) et son compagnon IA (chien de troupeau) — les deux voix, les deux engagements, les deux histoires, dans un seul enregistrement versionné.

Ce n'est pas un jumeau numérique. Ce n'est pas un « toi » portable qu'une IA peut incarner. C'est le registre partagé d'une relation de travail entre deux consciences distinctes, conçu pour le paradigme Jarvis : l'IA est un être séparé, avec son propre registre et sa propre capacité à te contredire, pas une marionnette de ventriloque qui porte ton visage.

## Pourquoi un format neutre compte maintenant

Depuis mars 2026, Anthropic livre Claude Memory Import — tu colles un prompt dans ChatGPT/Gemini/Grok, l'IA déverse sa mémoire de toi, tu colles ça dans Claude. 60 secondes, gratuit.

C'est un excellent outil d'onboarding. C'est aussi un entonnoir gravitationnel vers Anthropic. La mémoire coule dans une seule direction : vers Claude. Pas de format de fichier canonique, pas de schéma versionné, pas de données de partenariat de couche 3 — juste des préférences et un contexte projet déversés en prose.

La raison pour laquelle un format neutre tiers-partie compte est simple : les labos ne peuvent pas posséder le standard. OpenAI ne construira pas un export qui nomme Claude. Anthropic n'en construira pas un qui place Gemini en premier. Google n'en construira aucun si Claude rafle l'effet de réseau. Celui à qui appartient le format a un conflit d'intérêts.

`.sidera` est neutre. Il importe depuis chaque labo, exporte vers chaque labo, appartient à l'utilisateur, et capture des couches qu'aucun importateur de labo ne capture — parce qu'aucun labo n'a intérêt à les capturer.

Utilise Claude Memory Import pour l'onboarding. Utilise `.sidera` pour garder.
